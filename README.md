# Visuomotor Deep Optics

A comprehensive framework for learning visuomotor control policies for adaptive optics systems, exploring multiple approaches from supervised learning to optimization-enriched deep reinforcement learning.

## Overview

This project addresses the challenge of controlling adaptive optics systems through visual feedback, where the goal is to learn mappings from point spread function (PSF) observations to corrective actions that optimize optical performance. The framework implements and compares five distinct approaches, each with unique advantages and trade-offs for real-world deployment.

## Approaches

### 1. Direct Supervised Machine Learning (SML)
*Status: Planned*

Learn direct mappings from single PSF observations to perfect corrective actions.

**Implementation:**
- Sample observation-perfect_action pairs from the environment
- Train neural networks to map PSF images to actuator commands
- Use perfect_actions generated by simulated annealing as ground truth

**Pros:**
- ✅ Simple and straightforward implementation
- ✅ Fast inference once trained
- ✅ Well-understood training procedures
- ✅ Minimal data requirements per sample

**Cons:**
- ❌ Ground truth perfect actions not observable in real-world systems
- ❌ Distribution shift between training and deployment
- ❌ Bias accumulation across policy rollouts
- ❌ No ability to handle temporal dependencies
- ❌ Limited to single-step corrections

### 2. Simulated Annealing (SA) Optimization
*Status: ✅ Implemented and Working*

Direct optimization approach using simulated annealing to find optimal actions for any target PSF.

**Implementation:**
- Temperature-based stochastic search over action space
- Can target any desired PSF (not just "perfect")
- Generates perfect_actions and best_actions for imitation learning
- Located in `optomech/optimization/`

**Pros:**
- ✅ No ground truth required during data collection
- ✅ No prior knowledge of "perfect" solutions needed
- ✅ Can optimize for arbitrary target PSFs
- ✅ Provides theoretical optimality guarantees
- ✅ Handles complex, multimodal optimization landscapes

**Cons:**
- ❌ Extremely slow for real-world usage (seconds per action)
- ❌ Highly sensitive to environmental non-stationarity
- ❌ Requires stable environment during optimization
- ❌ Poor sample efficiency
- ❌ Cannot adapt to changing conditions in real-time

### 3. Behavioral Cloning / Imitation Learning (BC/IL)
*Status: 🚧 In Development*

Learn to imitate SA-generated trajectories through supervised learning on state-action sequences.

**Implementation:**
- Train on SA-generated episodes with perfect_action targets
- Learn temporal patterns and correction sequences
- Reduce dependency on environment stability
- Located in `optomech/imitation_learning/`

**Pros:**
- ✅ Faster than SA during inference
- ✅ Can learn complex multi-step correction strategies
- ✅ Leverages demonstrated optimal behavior
- ✅ More sample efficient than pure exploration
- ✅ Can incorporate temporal context

**Cons:**
- ❌ Still subject to distribution shift and bias accumulation  
- ❌ Limited exploration beyond demonstrated behaviors
- ❌ Not trained on temporal difference errors
- ❌ May struggle with novel situations
- ❌ Covariate shift problem during deployment

### 4. Deep Reinforcement Learning (DRL)
*Status: ✅ Implemented (TD3+LSTM+TBPTT)*

Learn through environmental interaction using temporal difference methods with recurrent networks.

**Implementation:**
- TD3 (Twin Delayed Deep Deterministic Policy Gradient)
- LSTM networks for handling partial observability
- Truncated Backpropagation Through Time (TBPTT)
- Located in `visuomotor-drl/`

**Pros:**
- ✅ Theoretically suitable for partially observable environments
- ✅ Can learn from sparse rewards and temporal credit assignment
- ✅ Adapts to environmental changes through continued learning
- ✅ Explores beyond demonstrated behaviors
- ✅ Handles temporal dependencies naturally

**Cons:**
- ❌ Challenging sample efficiency in practice
- ❌ High variance in learning dynamics
- ❌ Difficult visual representation learning from scratch
- ❌ Sensitive to hyperparameter choices
- ❌ May require extensive environment interaction

### 5. Optimization-Enriched DRL (OE-DRL)
*Status: 📋 Planned*

Bootstrap DRL training using BC-pretrained encoders and actors to reduce variance and accelerate learning.

**Implementation:**
- Initialize visual encoders using BC-trained networks
- Warm-start actor networks with BC policies
- Fine-tune using DRL for online adaptation
- Combine the stability of BC with the adaptability of DRL

**Pros:**
- ✅ Reduced variance compared to DRL from scratch
- ✅ Faster convergence through informed initialization
- ✅ Leverages both demonstrated and explored behaviors
- ✅ Better visual representation learning
- ✅ Maintains adaptability to environmental changes

**Cons:**
- ❌ Increased complexity in training pipeline
- ❌ Still requires careful balancing of BC and DRL components
- ❌ May inherit some biases from BC initialization
- ❌ Requires successful implementation of preceding approaches

## Getting Started

### Prerequisites

- Python 3.10.x (`>=3.10,<3.11`)
- [Poetry](https://python-poetry.org/) (package manager)
- Git

### Installation

**1. Clone the repository and switch to the active branch:**
```bash
git clone https://github.com/JustinFletcher/visuomotor-deep-optics.git
cd visuomotor-deep-optics
git checkout dev
```

**2. Install Python 3.10 (if not already available):**

Check your version:
```bash
python3 --version
```

If your system Python is below 3.10, install from source to a local prefix.
Python's standard library requires several C libraries that may not be present on minimal HPC systems.
Build all of them before building Python:

**libffi** (required for `_ctypes` — without it, `gymnasium` and many packages fail to import):
```bash
curl -LO https://github.com/libffi/libffi/releases/download/v3.4.6/libffi-3.4.6.tar.gz
tar -xzf libffi-3.4.6.tar.gz && cd libffi-3.4.6
./configure --prefix=$HOME/local
make && make install
cd ..
```

**bzip2** (required for `_bz2` — without it, `torchvision` and other packages fail to import):
```bash
curl -LO https://sourceware.org/pub/bzip2/bzip2-1.0.8.tar.gz
tar -xzf bzip2-1.0.8.tar.gz && cd bzip2-1.0.8
make -f Makefile-libbz2_so
make install PREFIX=$HOME/local
cp libbz2.so* $HOME/local/lib/
cd ..
```

**xz / liblzma** (required for `_lzma`):
```bash
curl -LO https://github.com/tukaani-project/xz/releases/download/v5.4.6/xz-5.4.6.tar.gz
tar -xzf xz-5.4.6.tar.gz && cd xz-5.4.6
./configure --prefix=$HOME/local
make && make install
cd ..
```

Then build Python, pointing it at all local libraries (note: some systems install to `lib64` — include both):
```bash
curl -LO https://www.python.org/ftp/python/3.10.14/Python-3.10.14.tgz
tar -xzf Python-3.10.14.tgz && cd Python-3.10.14
LDFLAGS="-L$HOME/local/lib -L$HOME/local/lib64" CPPFLAGS="-I$HOME/local/include" PKG_CONFIG_PATH="$HOME/local/lib/pkgconfig:$HOME/local/lib64/pkgconfig" \
  ./configure --prefix=$HOME/local --enable-optimizations
make -j$(nproc) && make install
export PATH=$HOME/local/bin:$PATH
echo 'export PATH=$HOME/local/bin:$PATH' >> ~/.bashrc
cd ..
```

Verify that all required modules built correctly before proceeding:
```bash
$HOME/local/bin/python3.10 -c "import _ctypes, _bz2, _lzma; print('All OK')"
```
If any module fails, its corresponding C library was not found during the build. Check both `$HOME/local/lib/` and `$HOME/local/lib64/` for the missing library, then rebuild Python with the correct paths.

> **HPC note:** Compute nodes often lack internet access. Download all source tarballs on a login node first — your home directory is typically shared across nodes.

On HPC systems, you may also be able to load a module:
```bash
module avail python
module load python/3.10   # if available
```

**3. Install Poetry:**
```bash
curl -sSL https://install.python-poetry.org | python3 -
export PATH=$HOME/.local/bin:$PATH
echo 'export PATH=$HOME/.local/bin:$PATH' >> ~/.bashrc
```

**4. Point Poetry to the correct Python (if you built from source):**
```bash
poetry env use $HOME/local/bin/python3.10
```

**5. Install project dependencies:**
```bash
poetry install
```

### Verify Setup
```bash
poetry run python utils/test_training.py mini_checkout_dataset
```

### Generate Training Data
```bash
# Generate dataset using job manager
poetry run python optomech/optimization/simple_job_manager_clean.py \
  --dataset_name my_dataset --target_transitions 1000
```

### Train Models
```bash
# Test training pipeline
poetry run python utils/test_training.py my_dataset

# Train behavioral cloning model (when implemented)
poetry run python optomech/imitation_learning/train_bc.py my_dataset
```
